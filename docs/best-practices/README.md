# Best Practices — Agents Reference

> Documentation de référence pour les agents smithers-workflows

## 📁 Structure

Chaque fichier `.md` contient les best practices spécifiques à un agent :

- **[frontend.md](./frontend.md)** ✅ — UX, Performance, Accessibilité (complet)
- **[backend.md](./backend.md)** 🚧 — API, BDD, Auth (à compléter)
- **[testing.md](./testing.md)** 🚧 — Unit, Integration, E2E (à compléter)
- **[review.md](./review.md)** 🚧 — Architecture, Qualité (à compléter)
- **[security.md](./security.md)** 🚧 — Vulnérabilités, OWASP (à compléter)

## 🎯 Utilisation

Les agents doivent :

1. **Lire** leur fichier de référence au démarrage
2. **Appliquer** les best practices dans leurs analyses et suggestions
3. **Citer** les sections pertinentes dans leurs commentaires
4. **Mettre à jour** ce fichier si de nouvelles pratiques sont découvertes

## 📝 Format

Chaque fichier suit la structure :

```markdown
# {Agent} Best Practices

## 🎯 Responsabilités
## 📋 Best Practices
### Catégorie 1
### Catégorie 2
## 🚨 Red Flags
## 📚 Ressources
```

## 🔄 Maintenance

- **Mise à jour** : Au fur et à mesure des recherches et découvertes
- **Validation** : Par le lead-dev avant merge
- **Versioning** : Date de dernière mise à jour en bas de chaque fichier
